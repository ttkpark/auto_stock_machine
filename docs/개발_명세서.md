## 2. 시스템 아키텍처 (4대 모듈)

```
main.py
  │
  ├── 1. brokers/          정보 수집기 (Data Collector)
  │     BaseBroker           증권사 API 인터페이스 (추상 클래스)
  │     MockBroker            모의투자 통신 (개발/테스트용)
  │     RealBroker            실전투자 통신 (실제 계좌)
  │
  ├── 2. analyzers/        AI 두뇌 (AI Analyzer)
  │     BaseAnalyzer          AI 분석기 인터페이스 (추상 클래스)
  │     GeminiAnalyzer        Google Gemini 연동
  │     ClaudeAnalyzer        Anthropic Claude 연동
  │
  ├── 3. utils/            판사 (Decision Maker)
  │     StockValidator        AI 환각 방지 (KRX 종목 검증)
  │     DecisionMaker         다수결 교차 검증 로직
  │
  └── 4. notifiers/        행동대장 (Executor)
        TelegramNotifier      텔레그램 알림 전송
```

### 핵심 설계 원칙: 플러그 앤 플레이 브로커

`BaseBroker` 추상 클래스를 기반으로 `MockBroker`(모의투자)와 `RealBroker`(실전투자)를 분리하여 구현합니다.

메인 봇 로직(`main.py`)은 브로커의 실제 구현체를 알 필요 없이 `BaseBroker` 인터페이스만 사용합니다. `.env` 파일의 `IS_REAL_TRADING` 값 하나만 변경하면 전체 환경이 전환됩니다.

```
IS_REAL_TRADING=False  →  MockBroker 사용 (모의투자)
IS_REAL_TRADING=True   →  RealBroker 사용 (실전투자)
```

## 3. 핵심 메커니즘

### 매수 (Buy) 메커니즘: "다수결 종목 발굴"

**실행 주기:** 평일 장 시작 전 08:30 (crontab 설정)

**흐름:**
```
1. broker.get_balance()          예수금 조회
2. analyzer.recommend_buy()      각 AI에게 매수 종목 추천 요청
3. DecisionMaker.find_buy_consensus()   다수결 합의 종목 선정
4. StockValidator.verify_and_get_code()  KRX 실제 상장 종목 검증 (환각 차단)
5. broker.get_current_price()    현재가 조회 → 매수 수량 계산
6. broker.buy_order()            매수 주문 실행
7. TelegramNotifier.notify_buy_order()  알림 전송
```

**매수 프롬프트 구조:**
```
당신은 한국 주식 전문가입니다.
내 주문 가능 예수금은 {balance}원입니다.
이 예수금 한도 내에서 지금 매수하면 좋을 한국 주식을 딱 1개만 추천해 주세요.
반드시 KRX에 실제 상장된 정확한 종목명을 사용해야 합니다.
답변은 반드시 아래 JSON 형식으로만 출력하세요.
{"종목명": "삼성전자", "이유": "저평가 구간 진입"}
```

### 매도 (Sell) 메커니즘: "개별 종목 정밀 진단"

**실행 주기:** 평일 장 마감 전 15:00 (crontab 설정)

**자동 조건 (AI 판단 없이 즉시 실행):**
- 손절: 수익률 -3% 이하 → 즉시 시장가 매도
- 수익률 +5% 이상 → AI에게 매도/보유 판단 요청

**매도 프롬프트 구조:**
```
당신은 한국 주식 전문가입니다.
내가 보유한 {종목명}({종목코드}) 주식은 {수량}주이고,
평단가는 {평단가}원인데 현재가는 {현재가}원 (수익률 {수익률}%)입니다.
현재 시장 상황에서 지금 매도할까요, 보유할까요?
답변은 반드시 아래 JSON 형식으로만 출력하세요.
{"결정": "매도", "이유": "목표 수익률 달성"}
```

### AI 환각(Hallucination) 방지 필터

AI가 존재하지 않는 주식 이름을 추천하는 경우를 차단합니다.

```
매일 아침 FinanceDataReader로 KRX 전체 상장 종목 딕셔너리 갱신
    ↓
AI 추천 종목명을 딕셔너리에서 검색
    ↓
존재함 → 종목코드 반환 → 매수 로직 진행
존재 안 함 → 즉시 중단 + 텔레그램 경고 알림
```

## 4. 매매 전략 파라미터

`config.py` 에서 조정 가능합니다.

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `BUY_BUDGET_RATIO` | 0.9 | 매수 시 사용할 예수금 비율 (90%) |
| `MIN_AI_CONSENSUS` | 2 | 매수 결정에 필요한 최소 AI 동의 수 |
| `TAKE_PROFIT_RATE` | 5.0 | AI 매도 판단 요청 기준 수익률 (%) |
| `STOP_LOSS_RATE` | -3.0 | 즉시 손절 기준 수익률 (%) |

## 5. 개발 체크리스트

### 1단계: 외부 API 및 권한 확보
- [ ] 한국투자증권 실전/모의투자 비대면 계좌 개설
- [ ] KIS Developers에서 App Key, App Secret 발급
- [ ] Google AI 스튜디오에서 Gemini API Key 발급
- [ ] Anthropic Console에서 Claude API Key 발급
- [ ] 텔레그램 BotFather로 봇 생성 및 Token 획득

### 2단계: 우분투 환경 구성
- [ ] Python 3.10 이상 설치 확인
- [ ] `pip install -r requirements.txt` 실행
- [ ] `.env.example` 복사 후 `.env` 파일 생성 및 키 입력
- [ ] `python main.py --mode status` 로 텔레그램 메시지 수신 확인

### 3단계: 모의투자 테스트
- [ ] `python main.py --mode buy` 로 매수 로직 단독 실행 테스트
- [ ] `python main.py --mode sell` 로 매도 로직 단독 실행 테스트
- [ ] KRX 환각 필터 동작 확인 (일부러 존재하지 않는 종목명 입력)
- [ ] 텔레그램 알림 모든 경우(매수/매도/오류) 정상 수신 확인

### 4단계: 자동화 (우분투 crontab)
- [ ] `crontab -e` 로 스케줄 등록
- [ ] 하루 동안 자동 실행 모니터링
- [ ] `logs/bot.log` 파일로 실행 기록 확인

### 5단계: 실전 전환
- [ ] 모든 테스트 완벽히 성공 후
- [ ] `.env` 파일에서 `IS_REAL_TRADING=True` 로 변경
- [ ] 최종 가동
